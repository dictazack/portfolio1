📖 Guide de déploiement étape par étape

Créer un repository GitHub

Allez sur github.com/new et créez un nouveau repository.

  • Nom: portfolio ou VOTRE_USERNAME.github.io
  • Visibilité: Public (requis pour GitHub Pages gratuit)
  • Ne pas initialiser avec README

Uploader les fichiers du portfolio

Option A: Interface web (plus facile)

  • Dans votre repository, cliquer sur "uploading an existing file"
  • Glisser-déposer TOUS les fichiers extraits du ZIP
  • Commit changes

Option B: Ligne de commande Git

cd portfolio-zakaria git init git add . git commit -m "Initial commit" git branch -M main git remote add origin https://github.com/VOTRE_USERNAME/portfolio.git git push -u origin main

Activer GitHub Pages

  • Aller dans Settings de votre repository
  • Menu de gauche: Pages
  • Source: Sélectionner GitHub Actions
  • Save

Vérifier le déploiement

  • Aller dans l'onglet Actions
  • Vous verrez un workflow en cours d'exécution
  • Attendre le badge vert ✅ (1-2 minutes)

Accéder à votre portfolio en ligne

Votre site est maintenant accessible à l'URL:

https://VOTRE_USERNAME.github.io/portfolio/
✅ Félicitations! Votre portfolio est en ligne et accessible partout dans le monde!

🔧 Dépannage

Le site ne se charge pas

  • Attendre 2-3 minutes après le premier déploiement
  • Vérifier que GitHub Pages est bien activé (Settings > Pages)
  • Vider le cache du navigateur (Ctrl + Shift + R)
  • Vérifier l'URL exacte dans Settings > Pages

CSS ou JavaScript ne fonctionnent pas

  • Vérifier que TOUS les fichiers ont été uploadés
  • Ouvrir la console du navigateur (F12) pour voir les erreurs
  • Vérifier que les chemins des fichiers sont corrects

Erreur dans GitHub Actions

  • Aller dans Actions > Workflow échoué > Voir les logs
  • Vérifier les permissions (Settings > Actions > Workflow permissions)
  • S'assurer que le fichier .github/workflows/deploy.yml existe
